## 🏗️ Architecture

* **Frontend (Vercel):** A Next.js/React application that interacts directly with the LLM and streams real-time responses using Server-Sent Events (SSE).
* **Backend AI Agent (Render):** A Python-based **FastMCP** server acting as an intelligent travel coordinator. It handles strict operational parameters, prevents LLM hallucinations, and provisions secure checkout links.
* **Database (Neon):** A Serverless PostgreSQL database containing comprehensive, real-time `airports` and `flights` tables.
* **Payment Gateway (Stripe):** Deep integration with Stripe Checkout to process secure payments directly generated by the AI agent's logic.

---

## ✨ Key Features

* 🗣️ **Natural Language Flight Search:** Converts conversational queries into exact database lookups using IATA codes, cities, or countries.
* ⚡ **Real-Time Streaming:** Provides a fast, interactive user experience via SSE.
* 🛡️ **Zero Hallucination Guarantee:** The agent is strictly sandboxed to only display flights present in the database, actively preventing fabricated options.
* 💳 **Secure AI Payments:** The LLM dynamically generates Stripe checkout sessions for a seamless end-to-end booking experience.
* ☁️ **Serverless Infrastructure:** Fully deployed across Vercel, Render, and Neon for high availability and zero maintenance overhead.

## 🧰 Available MCP Tools

The FastMCP server exposes the following tools to the LLM agent:

| Tool Name | Capability |
| :--- | :--- |
| **`search_flight_tool`** | Searches the Neon database for available flights based on user criteria (origin, destination, date, maximum budget). |
| **`payment_gateway`** | Generates a secure Stripe Checkout URL and presents it to the user in a non-clickable markdown block to preserve session integrity. |

## 🚀 Getting Started (Local Development)

### Prerequisites
* Python 3.10+
* A [Neon](https://neon.tech) PostgreSQL database instance
* A [Stripe](https://stripe.com) developer account

### 1. Clone the repository
```bash
git clone https://github.com/yourusername/dream-air-mcp.git
cd dream-air-mcp
```

### 2. Install dependencies
```bash
pip install -r requirements.txt
```

### 3. Environment Variables
Create a `.env` file in the root directory and configure the following keys:

```env
# Neon PostgreSQL connection string
DATABASE_URL=postgresql://user:password@hostname.neon.tech/dbname?sslmode=require

# Stripe Secret Key for checkout sessions
STRIPE_SECRET_KEY=sk_test_xxxxxxxxxxxxxxxxxxxxxxxx
```

### 4. Database Setup
Ensure your `airports.json` and `flightdata.json` files are located inside a `data/` folder one directory up (`../data/`), or adjust the file paths directly inside `db_helper.py`. 

Run the helper script to generate tables and load your data:
```bash
python db_helper.py
```

### 5. Run the MCP Server
Start the FastMCP server with SSE transport enabled:
```bash
python airline_booker.py
```
> **Note:** The server defaults to port `8000`. You can override this by setting a `PORT` environment variable before starting the application.

## 🤖 Local Testing with LLM Extensions

To test the MCP server locally using **Cline** (or an equivalent LLM IDE extension), add the following configuration to your MCP settings JSON file. 

> **Important:** Ensure the `command` and `args` paths correctly point to your local Python executable and project directory.

```json
{
  "mcpServers": {
    "airline-booking-server": {
      "command": "path/to/your/venv/bin/python", 
      "args": [
        "path/to/your/src/airline_booker.py"
      ],
      "env": {},
      "disabled": false
    }
  }
}
```
